# Тестирование

## Требования к тестированию

### Функциональные требования
1. **Авторизация:**
   - Регистрация нового пользователя
   - Вход в систему
   - Выход из системы
   - Восстановление пароля

2. **Управление контентом:**
   - Создание статьи
   - Просмотр статьи
   - Редактирование статьи
   - Удаление статьи
   - Список статей

### Нефункциональные требования
1. **Производительность:**
   - Время загрузки страницы < 2 секунды
   - Время сохранения статьи < 1 секунда

2. **Безопасность:**
   - Защита от CSRF-атак
   - Защита от XSS-атак
   - Проверка прав доступа

3. **Удобство использования:**
   - Интуитивно понятный интерфейс
   - Поддержка основных браузеров
   - Адаптивный дизайн

## Инструменты тестирования
- pytest (unit-тесты)
- Selenium (интеграционные тесты)
- coverage.py (покрытие кода)

## Запуск тестов

### Unit-тесты
```bash
python manage.py test cms_content
```

### Интеграционные тесты
```bash
pytest tests/integration/
```

### Проверка покрытия кода
```bash
coverage run manage.py test
coverage report
```

### Unit-тесты
- Модели: 100% покрытие
- Формы: 100% покрытие

### Интеграционные тесты
- Авторизация: пройдено

2. Нет обработки ошибок при сохранении файлов
